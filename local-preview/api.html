Callback log

    Callback contract

    What the host can listen to

    onLoadingChange
    Loading phase changes for red-light placeholders and runtime readiness.
    onReady
    Initial snapshot is available and PixiJS has mounted the canvas.
    onDriverOpen
    Host-specific navigation callback for opening a selected driver or project route.
    onDriverSelect
    User or host selected a driver.
    onRaceEvent
    Simulation events such as contact, safety-car, green-flag, lights-out, and race-finish.
    onLapChange
    Leader lap changed.
    onRaceFinish
    Winner and final classification are available.
    onError
    Runtime or host callback failures are reported without breaking the loop.

    Method contract

    What the returned controller can do

    restart
    Re-resolves options and starts a fresh run while preserving the mounted shell.
    selectDriver
    Focuses the camera, telemetry, overview, and race-data banner on a driver.
    setSafetyCarDeployed
    Deploys or clears safety-car mode; `callSafetyCar`, `clearSafetyCar`, and `toggleSafetyCar` are convenience wrappers.
    setRedFlagDeployed
    Switches red-flag state and refreshes race-control/pit-lane status.
    setPitLaneOpen
    Opens or closes pit entry while still respecting red-flag blocking.
    setPitIntent
    Requests or clears a driver pit stop with optional target compound.
    getPitIntent
    Reads the selected driver's numeric pit-intent state.
    getPitTargetCompound
    Reads the target compound stored by the pit strategy request.
    servePenalty
    Marks a serviceable drive-through or stop-go penalty as served.
    cancelPenalty
    Cancels a penalty and clears its time/service effects.
    getSnapshot
    Returns the current race snapshot used by host dashboards and logs.
    destroy
    Stops the app and empties the mounted root.
