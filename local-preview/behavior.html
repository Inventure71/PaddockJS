Package Contracts

Behavior a host can rely on

These examples explain the sizing, loading, banner, timing, and race-finish rules that make PaddockJS reusable outside this preview.

Timing fit

Expand versus scroll

`expand-race-view` grows the race window to fit the timing tower. `scroll` keeps a fixed window and scrolls timing rows inside the tower.

expand-race-viewtimingTowerVerticalFit: 'expand-race-view'
scrolltimingTowerVerticalFit: 'scroll'

Race-data banners

Auto and custom sizing

`auto` uses the open race space beside the timing board and overlaps only when needed. `custom` preserves the host-tunable lower-third geometry.

initial
`project`, `radio`, or `hidden` chooses the first lower-third state.
enabled
Controls which banner types may appear after startup and driver selection.
auto
Package chooses whether to sit beside or over the timing tower.
custom
Host keeps explicit lower-third sizing through package CSS variables.

Theme and sizing

Stable CSS-variable contract

Prefer the public `theme` object over host CSS overrides. It keeps package geometry centralized.

accentColor--paddock-accent-color
greenColor--paddock-green-color
yellowColor--paddock-yellow-color
timingTowerMaxWidth--paddock-timing-tower-max-width
raceViewMinHeight--paddock-race-view-min-height

Race lifecycle

Finish, winner, classification

The winner is recorded when the first car finishes. The race keeps running until every car is classified, then the field circulates under safety car.
