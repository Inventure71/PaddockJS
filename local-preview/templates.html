Template Presets

Prebuilt simulator shells

These examples use `mountF1Simulator()`. A host can start from a preset and override only the fields it actually owns.

complete-broadcast

Race workbench with built-in timing, telemetry drawer, external controls, lower-third messages, top steward messages, and timing penalty badges.

dashboard

Default full shell: timing tower, race canvas, telemetry, overview, and lower-third.

timing-overlay

Timing tower lives inside the race view with the adaptive lower-third and external camera controls.

compact-race

Race-first mount for small embeds, with fewer surrounding surfaces.

full-dashboard

Dense package dashboard with telemetry and timing visible for repeated inspection.

race drawer

Canvas, timing tower, and the project lower-third stay together while telemetry opens into a right drawer.

Template: complete-broadcast

Complete race workbench

One package-owned race view with the timing tower, telemetry drawer, project/radio lower-third, and penalty UI enabled for normal racing incidents.

const simulator = createPaddockSimulator({
  rules: { ruleset: 'custom', modules: { penalties: { trackLimits: { strictness: 1 } } } },
  ui: {
    penaltyBanners: true,
    timingPenaltyBadges: true,
    raceDataBanners: { initial: 'project', enabled: ['project', 'radio'] },
  },
})

mountRaceTelemetryDrawer(root, simulator, {
  raceDataTelemetryDetail: true,
  timingTowerVerticalFit: 'expand-race-view',
})

Preset: dashboard

Standard full shell

The complete package UI. Use this when the simulator is the page’s primary experience.

mountF1Simulator(root, { preset: 'dashboard', drivers, entries })

Preset: timing-overlay

Tower inside race view

The timing tower, camera safe area, and race-data banner are owned by the package layout.

mountF1Simulator(root, { preset: 'timing-overlay', ui: { raceDataBanners: { initial: 'radio' } } })

Option: radio/project banner

Project lower-third with sector detail

The same lower-third can start as project radio, switch to a selected project, and show compact S1/S2/S3 detail without mounting the standalone sector popup.

mountRaceCanvas(root, simulator, {
  includeTimingTower: true,
  includeRaceDataPanel: true,
})

createPaddockSimulator({
  ui: {
    raceDataBanners: { initial: 'radio', enabled: ['project', 'radio'] },
    raceDataTelemetryDetail: true,
  },
})

Preset: compact-race

Small host embed

Compact mode keeps the simulator usable when the host gives it less vertical space.

mountF1Simulator(root, { preset: 'compact-race', theme: { raceViewMinHeight: '520px' } })

Preset: full-dashboard

Inspector shell

Full dashboard mode keeps the control surface broad while still allowing host theme overrides.

mountF1Simulator(root, { preset: 'full-dashboard', theme: { accentColor: '#00ff84' } })

Template: race telemetry drawer

Race window with telemetry drawer

Canvas, timing tower, and the project banner mount together. The telemetry button opens a package-owned sidebar that takes layout space from the race window while the safety-car control stays available.

mountRaceTelemetryDrawer(root, simulator, {
  timingTowerVerticalFit: 'expand-race-view',
  raceDataTelemetryDetail: true,
})
