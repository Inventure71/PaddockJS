Composable API

Package-owned pieces in host layout

`createPaddockSimulator()` lets host pages arrange the simulator while PaddockJS still owns markup, CSS variables, event binding, and runtime state.

Race window component

Canvas, timing tower, and banner together

This mount demonstrates the reusable race-window package: canvas, timing tower, start lights, lower-third, top steward messages, and timing penalty badges in one root.

mountRaceCanvas(root, simulator, {
  includeTimingTower: true,
  includeRaceDataPanel: true,
  timingTowerVerticalFit: 'expand-race-view',
})

ui: {
  penaltyBanners: true,
  timingPenaltyBadges: true,
}

Separate mounts

Every public surface mounted independently

The host controls placement only. The mounted roots below cover every public composable mount exported by the package and bind them to shared controller state where appropriate.

Race controlsmountRaceControls()

Header, safety toggle, and restart button.

Safety carmountSafetyCarControl()

Standalone control wired to the same race state.

Camera controlsmountCameraControls()

External camera mode and zoom controls.

Timing towermountTimingTower()

Fixed-width broadcast list with top-stacked rows and internal scrolling.

Race canvasmountRaceCanvas()

Canvas-only mount, useful when the host places timing or telemetry elsewhere.

Core telemetrymountTelemetryCore()

Scalar selected-car readouts without owning the graph or tables.

Sector graphmountTelemetrySectors()

Live S1/S2/S3 progress as its own optional component.

Sector bannermountTelemetrySectorBanner()

Broadcast lower-third sector graph, matching the radio/data banner placement style.

Telemetry stackmountTelemetryPanel()

Combined telemetry mount with core readouts, sectors, lap timing, sector timing, and the optional overview variant enabled.

Lap tablemountTelemetryLapTimes()

Current, last, and best lap timing in a separate table.

Sector tablemountTelemetrySectorTimes()

Last and best sector times detached from the core telemetry readouts.

Car and driver overviewmountCarDriverOverview()

Center visual plus linked cells for car fields or driver fields.

Race data panelmountRaceDataPanel()

Standalone lower-third for hosts that intentionally want it outside the race canvas.

Telemetry drawermountRaceTelemetryDrawer()

Composite race workbench variant with camera controls, safety-car control, race canvas, timing tower, lower-third, simulation-speed toggle, and an open telemetry drawer.
