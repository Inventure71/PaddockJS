## Definition Of Done

- Large mixed-responsibility modules are either split or intentionally documented as facade/orchestrator files.
- Canonical module paths are clear; compatibility re-export files exist only where package contracts require them.
- Simulation, rendering, environment, UI template, and app runtime responsibilities are separated by feature-owned modules.
- No obvious duplicate/dead code remains from the cleanup passes.
- Public API behavior is preserved.
- Determinism is preserved for the same seed, track seed, drivers, entries, and rules.
- `docs/architecture.md`, `docs/rules.md`, `docs/concepts.md`, `docs/data_contract.md`, and `README.md` are updated only where ownership or behavior docs actually changed.
- Linear is updated with the completed work and verification evidence.
